# 🧠 Object Detection 기반 비정상 주행(Driving-Abnormal) 판별 로직 설계

본 프로젝트에서는 단순히 이미지를 통째로 분류하는 한계를 넘어, 객체 탐지(YOLO) 모델이 반환하는 **Bounding Box의 좌표, 종류(Class), 기하학적 관계**를 종합적으로 분석하여 비정상 주행 상황을 정밀하게 판별하는 룰베이스 파이프라인을 설계하였습니다.

탐지 대상 클래스는 총 4가지(`human`, `kickboard`, `helmet`, `bare_head`)이며, 도로 환경에서 발생할 수 있는 오진율을 최소화하기 위해 단계별로 알고리즘을 업그레이드 했습니다.

---

### 📍 1차 안: 이미지 내 단순 객체 총 개수 비교 (Baseline)
* **개념:** 단일 이미지 내에서 탐지된 `human`과 `kickboard` 전체 개수를 카운트하여 단순 비교하는 방식.
* **판단 규칙:** $$\text{human Count} > \text{kickboard Count} \implies \text{Abnormal (다인승)}$$
* **한계점:** 킥보드 탑승자 외에 **배경에 찍힌 일반 행인**까지 모두 `human`으로 감지되므로, 혼잡한 도로나 인도 환경에서 정상 주행을 비정상으로 오진하는 오류 발생 가능성(False Positive)이 매우 높음.

---

### 📍 2차 개선안: 공간 좌표(Y축 원근법) 기반 행인 필터링
* **개념:** 1차 안의 행인 오감지 문제를 해결하기 위해, 킥보드와 탑승자는 물리적으로 동일한 수직선상에 밀착해 있다는 특성을 활용.
* **상세 로직:**
  * **X축 중심점 대조:** 특정 `kickboard` 박스의 X축 중심 영역을 기준으로, 좌우 범위를 벗어난 일반 행인을 1차 필터링.
  * **Y축 바닥 좌표(Distance) 비교:** 원근법으로 인해 멀리 있는 행인은 Y축 좌표가 다르게 찍힘. 킥보드 하단(발판) 좌표와 사람 하단(발) 좌표 사이의 거리를 계산하여, 기준치 이상 떨어진 객체는 탑승자 후보군에서 자동 제외.
* **한계점:** 이미지마다 scale이 다르기 때문에 킥보드 하단 좌표와 사람 하단 좌표에 대해 고정된 기준치를 정하는 것이 어려움. 다인승의 경우 발 위치에 대한 차이로 인한 오류 가능성도 존재.

---

### 📍 3차 최종안: 인접도 연산 및 탑승자 간 IoU 복합 판별
* **개념:** 2차 안의 한계를 극복하기 위해, 킥보드와 사람의 관계뿐만 아니라 **좁은 공간에 밀착한 탑승자 간의 상호 관계**까지 복합적으로 연산하는 최종 알고리즘.
* **상세 로직:**
  * **바닥 타이트 매칭 (Tight Boundary Match):** `human` 박스의 하단 좌표($Y_{\text{max}}$)와 `kickboard` 박스의 상단~중단 좌표 사이의 절대적 마진(Margin)을 계산하여 공중에 떠 있거나 멀리 있는 행인을 완벽 차단.
  * **Rider-to-Rider IoU:** 한 대의 킥보드 발판 위에 2명 이상이 겹쳐 타면, **사람 박스와 사람 박스 간의 IoU(겹침 비율)가 비정상적으로 높아진다**는 점을 이용.

---

### 🗂️ 🚨 종합 비정상(Abnormal) 상황 판단
위의 3차 매칭 알고리즘을 거쳐 **특정 킥보드에 탑승한 것으로 최종 확정된 탑승자 그룹**이 식별되면, 다음 3가지 핵심 규칙을 순차적으로 적용하여 최종 주행 상태를 판별합니다.

| 판별 항목 | 데이터 조건 (Rule) | 최종 상태 |
| :--- | :--- | :--- |
| **① 다인승 탑승** | 한 대의 킥보드 영역에 매칭된 `human` $\ge 2$ 이며, 탑승자 간 IoU가 임계치 이상일 때 | **Abnormal (위험 주행)** |
| **② 헬멧 미착용** | 킥보드 탑승자(`human`)는 존재하나, 해당 탑승자 상단 영역 내에 `helmet`이 탐지되지 않을 때 | **Abnormal (위험 주행)** |
| **③ 정상 주행** | 위의 세 가지 비정상 조건에 해당하지 않는 안전한 1인 탑승 상태일 때 | **Normal (정상 주행)** |
